1. 기본 정보
--------------------------------------------------------------------------------

Base URL (로컬)
  http://localhost:4000/api

Base URL (배포)
  https://{배포-도메인}/api

Content-Type
  application/json

인증 헤더 (대부분의 엔드포인트)
  Authorization: Bearer <JWT>

JWT 발급
  POST /api/auth/register 또는 POST /api/auth/login 응답의 token 필드

사용자 역할 (role)
  user | company | institution | admin

공통 오류 응답
  400  Validation failed (Zod issues 포함)
  401  Authentication required / Invalid or expired token
  403  Insufficient role permission / 프로젝트 권한 부족
  404  Route not found / 리소스 없음
  409  Email is already registered 등
  500  Internal server error

참고 (외부 연동 시)
  - 파트너용 API 키, OAuth, OpenAPI, API 버전(/v1), Rate limit, Webhook 은 미구현
  - CORS는 FRONTEND_URL(및 FRONTEND_URLS)에 등록된 출처만 허용
  - 이미지는 data URL(base64) 또는 https URL 형태로 전달

3. 인증 (Auth) — /api/auth
================================================================================

POST /register
     회원가입 → JWT + user 반환
     Body: { email, password (min 8), name (2~80), role? (기본 user) }

POST /login
     로그인 → JWT + user 반환
     Body: { email, password }

POST /logout
     로그아웃 (클라이언트에서 토큰 삭제용, 서버 토큰 무효화 없음)

GET  /me
     [인증] 현재 로그인 사용자 정보

5-1. 검색 (Search) — /api/search
================================================================================

[인증 필요]

GET   /recommended
      추천 친구 (비슷한 커리어·자주 본 프로필·프로젝트 팀·네트워크·관심 글 참여)
      Query: limit? (1~30, 기본 12)
      응답: { results[] } — matchReasons 예: 비슷한 커리어, 자주 본 프로필, 같은 프로젝트 등
      이미 팔로우 중인 사용자·본인 제외

GET   /
      사람·프로필 검색 (이름, 직무 headline, 회사, 학교, bio, 스킬, 경력)
      Query: q (1~80자, 필수), limit? (1~30, 기본 20)
      응답: { query, results[] }
      results 항목: userId, name, avatarUrl, role, headline, company, school,
                    skills (최대 6), matchReasons (이름|직무|회사|스킬|학교)
      본인(userId)은 결과에서 제외

프로필 조회 기록: GET /api/profiles/:userId 시 타인 프로필이면 ProfileView에 누적 (추천 반영)

6. 프로젝트 (Projects) — /api/projects
================================================================================

[전체 인증 필요]
프로젝트 내부 동작은 RBAC(역할·권한)에 따라 403 가능

--- 목록·생성 ---

GET   /
      내가 참여 중이거나 공개된 프로젝트 목록

POST  /
      프로젝트 생성

--- 초대 (전역 경로) ---

POST  /invites/:inviteId/accept
      프로젝트 초대 수락

POST  /invites/:inviteId/decline
      프로젝트 초대 거절

--- 권한 카탈로그 ---

GET   /permission-catalog
      프로젝트에서 사용 가능한 권한 목록 (16종)

--- 단일 프로젝트 ---

GET    /:id
       프로젝트 상세
       비멤버 + 공개 프로젝트: 요약만 (tasks/files/activities/roles 미포함, isGuestViewer)

PATCH  /:id
       프로젝트 수정 (권한: project.edit 등)

DELETE /:id
       프로젝트 삭제 (권한: project.delete)

--- 팀 ---

GET    /:id/team-candidates
       초대 가능한 사용자 후보 (팔로우 관계 기반)

POST   /:id/invites
       팀원 초대 (권한: members.invite)

POST   /:id/members
       멤버 직접 추가

DELETE /:id/members/me
       프로젝트에서 나가기

--- 태스크 ---

POST   /:id/tasks
       태스크 생성 (권한: tasks.create)

PATCH  /:id/tasks/:taskId
       태스크 수정 (권한: tasks.edit)

--- 활동·파일 ---

POST   /:id/activities
       활동 기록 추가 (권한: activities.create)

POST   /:id/files
       파일 메타 추가 (권한: files.upload)

--- 역할(RBAC) ---

GET    /:id/roles
       프로젝트 역할 목록

POST   /:id/roles
       역할 생성 (권한: roles.manage)

PATCH  /:id/roles/:roleId
       역할 수정 (권한: roles.manage)

DELETE /:id/roles/:roleId
       역할 삭제 (권한: roles.manage, 시스템 역할 제외)

PATCH  /:id/members/:memberId/role
       멤버 역할 변경 (권한: members.assign_role)

프로젝트 권한 ID 목록 (permission-catalog / 역할 설정 시 사용)
  project.view, project.edit, project.delete, project.settings
  roles.manage
  members.invite, members.remove, members.assign_role
  tasks.view, tasks.create, tasks.edit, tasks.delete
  files.view, files.upload, files.delete
  activities.create

기본 시스템 역할 (프로젝트 생성 시 시드)
  owner, admin, member, viewer

================================================================================

       전체 피드

       추천 피드 (관심·스킬·프로젝트 기반)

GET    /by-user/:userId
       특정 사용자 게시물

GET    /me
       내 게시물

POST   /
       게시물 작성 (텍스트, 이미지, 프로젝트 연결, 공개범위, 리포스트 등)

GET    /public/by-user/:userId
       사용자 공개 게시물 목록 [인증 불필요]

GET    /public/:id
       공개 게시물 상세 [인증 불필요, PUBLIC만]

GET    /:id
       게시물 상세

PATCH  /:id
       게시물 수정 (작성자만)

DELETE /:id
       게시물 삭제 (작성자만)

POST   /:id/like
       좋아요

DELETE /:id/like
       좋아요 취소

POST   /:id/comments
       댓글 작성 (body: content, 선택 parentId — 최상위 댓글에만 대댓글 가능)

POST   /:id/comments/:commentId/like
       댓글 좋아요

DELETE /:id/comments/:commentId/like
       댓글 좋아요 취소

게시물·댓글 응답의 likes[]: id, userId, createdAt, user { id, name, avatarUrl }

8. 기회 (Opportunities) — /api/opportunities
================================================================================

[전체 인증 필요]
생성·수정·삭제는 role 이 admin 또는 institution 만 가능

GET    /
       기회 목록 (필터·매칭 점수 포함)

POST   /
       기회 생성 [admin, institution]

GET    /enrollments/me
       내 신청(수강) 목록

GET    /enrollments/managed
       기관·관리자가 관리하는 수강 목록

PATCH  /enrollments/:enrollmentId
       수강 상태 변경 (승인·거절 등)

GET    /:id
       기회 상세

PATCH  /:id
       기회 수정 [admin, institution]

DELETE /:id
       기회 삭제 [admin, institution]

POST   /:id/enroll
       프로그램 신청 (취소·거절 후 재신청 시 APPLIED로 갱신)

DELETE /:id/enroll
       프로그램 신청·수강 취소 (본인, APPLIED·ENROLLED → WITHDRAWN)

POST   /:id/save
       기회 저장(북마크)

DELETE /:id/save
       저장 취소
